### Chroma

The `Chroma` is a component designed for implementing a Vector Store using Chroma. This component allows users to utilize Chroma for efficient vector storage and retrieval within their language processing workflows.

For detailed documentation and integration guides, please refer to the [Chroma Component Documentation](https://python.langchain.com/docs/integrations/vectorstores/chroma).


**Configuration Options**

The `Chroma` offers various configuration options to customize its behavior. Here are the available options:

- **Collection Name**: The name of the collection.
- **Persist Directory**: The directory to persist the Vector Store to.
- **Code**: Advanced option for code settings.
- **Input**: Specifies the type of input (Document or Record).
- **Embedding**: Specifies the embedding to use for the Vector Store.
- **Server CORS Allow Origins**: CORS allow origins for the Chroma server.
- **Server Host**: The host for the Chroma server.
- **Server Port**: The port for the Chroma server.
- **Server gRPC Port**: The gRPC port for the Chroma server.
- **Server SSL Enabled**: Whether to enable SSL for the Chroma server.

**Methods**

****`build_config()`****

Builds the configuration for the component.

**Returns**

- `dict`: A dictionary containing the configuration options for the component.

****`build()`****

Builds the Vector Store or BaseRetriever object.

**Arguments**

- `collection_name (str)`: The name of the collection.
- `index_directory (Optional[str])`: The directory to persist the Vector Store to.
- `chroma_server_ssl_enabled (bool)`: Whether to enable SSL for the Chroma server.
- `embedding (Optional[Embeddings])`: The embeddings to use for the Vector Store.
- `inputs (Optional[List[Record]])`: The inputs to use for the Vector Store.
- `chroma_server_cors_allow_origins (Optional[str])`: The CORS allow origins for the Chroma server.
- `chroma_server_host (Optional[str])`: The host for the Chroma server.
- `chroma_server_port (Optional[int])`: The port for the Chroma server.
- `chroma_server_grpc_port (Optional[int])`: The gRPC port for the Chroma server.

**Returns**

- `Union[VectorStore, BaseRetriever]`: The Vector Store or BaseRetriever object.

### Weaviate Search

The `WeaviateSearch` component facilitates searching a Weaviate Vector Store for similar documents.

**Configuration**

The `WeaviateSearch` component can be configured using the following parameters:

- **Search Type**: Specifies the type of search operation to perform. Options include "Similarity" and "MMR".
- **Input**: The input text utilized for the search operation.
- **Weaviate URL**: URL of the Weaviate server. Default is "http://localhost:8080".
- **API Key**: (Optional) API key required for authentication with the Weaviate server.
- **Index Name**: (Optional) Name of the index to search within the Weaviate server.
- **Text Key**: (Optional) Key representing the text field in the indexed documents. Default is "text".
- **Embedding**: The embedding used for representing text documents.
- **Attributes**: (Optional) Additional attributes considered during the search operation.
- **Search By Text**: (Optional) When enabled, the search operation is performed based on the input text.
- **Code**: (Hidden) This parameter is not visible in the configuration.

**Build Method**

The `build` method of the `WeaviateSearchVectorStore` component constructs the search functionality. It accepts the following parameters:

- **input_value**: The input text for the search operation.
- **search_type**: Specifies the type of search operation, either "Similarity" or "MMR".
- **url**: The URL of the Weaviate server.
- **search_by_text**: (Optional) If set to `True`, the search operation is performed based on the input text.
- **api_key**: (Optional) API key for authentication with the Weaviate server.
- **index_name**: (Optional) Name of the index to search within the Weaviate server.
- **text_key**: (Optional) Key representing the text field in the indexed documents.
- **embedding**: The embedding used for representing text documents.
- **attributes**: (Optional) Additional attributes considered during the search operation.

**Return Value**

The `build` method returns a list of records containing documents similar to the input text based on the specified search type. If the Weaviate index fails to load, a `ValueError` is raised.
